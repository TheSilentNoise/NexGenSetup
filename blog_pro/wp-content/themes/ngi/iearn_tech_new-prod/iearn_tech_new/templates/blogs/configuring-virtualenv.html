Configuring Virtual Environment

What is virtualenv?

Virtualenv is a tool that enables developers to create an isolated python environment for each project.

It enables developers to install required packages in the particular project folder that will be managed by virtualenv

 

What is the advantage of using virtualenv?

Different projects may need different set and version of packages which and the versions may have interdependencies.

It becomes hard to maintain if you install packages globally.

The advantage of using virtualenv is that it creates isolated environments for each project and keeps the global ‘site-packages’ folder clean.

 

Installation

pip install virtualenv

 

Initialize virtualenv for your project

cd <project_directory>

virtualenv –no-site-packages venv

 

This will create a venv subdirectory in your project where everything is installed

 

Activate virtualenv for your project

Source venv/bin/activate

(venv) will appear at the beginning of your terminal prompt indicating that you are working inside the virtualenv. Now when you will install something like

pip install <requirements.txt>

It will get installed inside venv folder, and not conflict with other projects in the system.

 

Exit

To exit the virtual environment, you should run

deactivate
