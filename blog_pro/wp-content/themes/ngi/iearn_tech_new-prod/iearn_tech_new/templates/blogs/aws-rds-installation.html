AWS RDS Installation and Configuration

AWS RDS

RDS (Relational Database Service) is a database service on cloud platform. It is a web-service which is easier to setup, operate and scale a relational database in the cloud.

It is cost-efficient, simple to setup, and can scale the relational database for the applications. Administrative tasks like backing up of database, recovery are managed automatically by RDS.


Open the security group for EC2

Add a new rule for MySQL/Aurora, for opening the port : 3306 , which can be accessed by anyone.

Select the service RDS from the Services after logging in to the AWS

Click on Get Started Now



Select Engine (Free Tier Support only)

MySQL

Specify DB Details

Select the DB Engine version as per the need ( in our case it was mysql 5.6.7)

DB Instance Class – db2.t2.micro – 1vCPU, 1GiB RAM ( as per free tier)

Allocated Storage – 20 GB

Storage Type – General Purpose (SSD)

Multi-AZ Deployment – No ( Set to Yes, the DB instance can be replicated to other availability zones, if needed)

DB Instance Identifier – iearntech

Master username – root

Master password – Enter your master password

Configure advanced settings –

Virtual Private Cloud – Default VPC

Subnet Group – default

Public Accessibility – Yes( Whether this particular DB Instance can be connected from EC2 instances and devices outside of the selected VPC)

Availablity Zone –us-west-1a

Database Name – iearn_db

Database port – 3306

DB Parameter Group – default.mysql5.6

Option Group – default:mysql-5-6

IAM DB Authentication – disable

Security Group – Select all the groups

And leave all the other settings as it is.

Open Workbench

Host Name – Endpoint of RDS Instance

Port – 3306
